後台 面板 網站

  • 2016-07-02
  • Ming yu
  • web

[前言...]
剛進入公司時，公司採用bootstrap 建立簡單基本的後台。但是想要將後台改得較好使用。因為一些時間及成本上的考量，且希望將設計的時間使用在前臺畫面設計以及互動上。因此主管決定購買順眼的面板後，由我負責套上面板，改進後台。
Ruby On Rails 在套面板有時候比較麻煩，為了節省時間。通常我都直接放在public資料夾。避免相對路徑錯誤。
基本上面板的文件都很清楚大多數功能也都有，我只要負責套上面板後，再額外客制些功能即可。

某個網站的後台面板


由於公司是接案公司，會同時替很多客戶服務。為了在客戶發現錯誤，或是紀錄網站錯誤方便查找問題。我就架了一個網站專門接受個網站出錯的訊息。當其他網站出現錯誤時，就會從主機發送訊息到我架的錯誤收集網站。這樣我就可以主動的去修改bug。
